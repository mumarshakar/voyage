Step 3: Add @font-face Declarations

üé® CSS Setup: We need to tell the browser where to find your font files using @font-face declarations. This goes in your theme.css file.

3.1 Open theme.css File

‚ñº
  1. In the Assets folder, click on theme.css
  2. The file will open in the code editor
  3. Scroll to the top of the file (or find a good location after the existing CSS)
‚ö†Ô∏è Best Practice: Add @font-face declarations at the top of your CSS file, before any other styles. This ensures fonts load early.

3.2 Add @font-face for Regular Weight

‚ñº

Copy and paste this code at the top of your theme.css file:

/* GP Alpina Font - Regular Weight */ @font-face { font-family: 'GP Alpina'; src: url('{{ "gp-alpina-regular.woff2" | asset_url }}') format('woff2'), url('{{ "gp-alpina-regular.woff" | asset_url }}') format('woff'); font-weight: 400; font-style: normal; font-display: swap; }
üìù Explanation:
  • font-family: 'GP Alpina' - Name you'll use to reference the font
  • src: url(...) - Shopify Liquid syntax to load assets
  • format('woff2') - Tells browser the file format
  • font-weight: 400 - Regular weight
  • font-display: swap - Better loading performance

3.3 Add @font-face for Bold Weight

‚ñº

Add this code right after the Regular weight declaration:

/* GP Alpina Font - Bold Weight */ @font-face { font-family: 'GP Alpina'; src: url('{{ "gp-alpina-bold.woff2" | asset_url }}') format('woff2'), url('{{ "gp-alpina-bold.woff" | asset_url }}') format('woff'); font-weight: 700; font-style: normal; font-display: swap; }
üí° Note: If you have other weights (Light 300, Medium 500, etc.), add them with their respective font-weight values.

3.4 Add Italic Variants (If Available)

‚ñº

If you have italic versions, add them like this:

/* GP Alpina Font - Regular Italic */ @font-face { font-family: 'GP Alpina'; src: url('{{ "gp-alpina-italic.woff2" | asset_url }}') format('woff2'), url('{{ "gp-alpina-italic.woff" | asset_url }}') format('woff'); font-weight: 400; font-style: italic; font-display: swap; } /* GP Alpina Font - Bold Italic */ @font-face { font-family: 'GP Alpina'; src: url('{{ "gp-alpina-bold-italic.woff2" | asset_url }}') format('woff2'), url('{{ "gp-alpina-bold-italic.woff" | asset_url }}') format('woff'); font-weight: 700; font-style: italic; font-display: swap; }

4.4 Update Font Helper in theme.liquid

‚ñº

Now we need to tell the theme how to use GP Alpina. Open layout/theme.liquid:

  1. Navigate to layout folder
  2. Open theme.liquid
  3. Find the case statements that convert font family values to CSS

Find the heading_font_family case statement and add:

case heading_font_family when 'gp-alpina' assign heading_font_family_css = "'GP Alpina', sans-serif" when 'playfair' assign heading_font_family_css = "'Playfair Display', 'Times New Roman', serif" ...

Do the same for subheading_font_family:

case subheading_font_family when 'gp-alpina' assign subheading_font_family_css = "'GP Alpina', sans-serif" when 'playfair' assign subheading_font_family_css = "'Playfair Display', 'Times New Roman', serif" ...

And for body_font_family:

case body_font_family when 'gp-alpina' assign body_font_family_css = "'GP Alpina', sans-serif" when 'playfair' assign body_font_family_css = "'Playfair Display', 'Times New Roman', serif" ...
‚úì Save All Files: Make sure to save both settings_schema.json and theme.liquid

5.4 Troubleshooting

‚ñº

Font not showing? Check these common issues:

  • File names don't match: Make sure the file names in @font-face match exactly what you uploaded
  • Wrong file path: Verify the asset_url filter is correct
  • Browser cache: Hard refresh (Ctrl+F5 or Cmd+Shift+R) to clear cache
  • Font not loaded: Check browser DevTools ‚Üí Network tab to see if fonts are loading
  • Syntax errors: Verify your CSS and JSON files have no syntax errors
üêõ Debug Tip: Open browser DevTools (F12), go to Console tab, and look for any font loading errors.
